1What is Graph Coloring?

Graph coloring assigns a color to each vertex so that neighboring vertices never share a color. This appears in register allocation, exam scheduling, and frequency assignment.

Key Concept

A coloring is a function c: V -> {1,2,3,...}. A graph is properly colored when every edge (u,v) satisfies c(u) != c(v). The smallest number of colors needed is chi(G).

Example

In a 5-cycle, three colors are enough: color around the cycle with 0,1,2,0,1. Any attempt with only two colors fails because an odd cycle forces a conflict.

Check Understanding

If two vertices are not adjacent, they may share a color. Only edges impose constraints.

Try it yourself: Playground -> preset C5 -> Build Steps -> Step.

2The 3-Coloring Problem

The decision problem asks: given graph G, does there exist a proper coloring using at most 3 colors? This threshold is where complexity changes sharply.

Formal Statement

3-COL = { G | chi(G) <= 3 }. Output is YES/NO, not the minimum color count.

Important Distinction

2-coloring is polynomial (bipartite check), but 3-coloring is NP-complete.

Quick Test

K4 is not 3-colorable because all four vertices are pairwise adjacent and require four distinct colors.

Try it yourself: Playground -> preset K4 -> set algorithm to Backtracking (k=3) -> Build Steps -> Play.

3Why It's Hard — NP-Completeness

A problem is in NP when a proposed solution can be verified quickly. For 3-coloring, a certificate is simply a color assignment for each vertex.

Verifier Runtime

Scan every edge once and reject if endpoints have equal colors. This is O(|V|+|E|) time, so 3-coloring is in NP.

for each edge (u, v) in E: if color[u] == color[v]: reject accept
Do Not Over-Claim

NP-complete does not prove impossible; it means no polynomial-time algorithm is known unless P = NP.

Try it yourself: NP-Completeness tab -> Verifier panel -> assign colors -> Verify.

4Reduction from 3-SAT

To prove NP-hardness, we map any 3-CNF formula phi to a graph G_phi so that phi is satisfiable if and only if G_phi is 3-colorable.

Reduction Contract

Palette triangle enforces colors T,F,B, variable gadgets force each variable to choose exactly one of T/F, and clause gadgets enforce OR behavior.

Clause Gadget (Spec-aligned)

Each clause adds internal vertices x,y,z,m,n,o with fixed edges and anchor o-F. This is the core satisfiability constraint.

Literal links: a-x, b-y, c-z Internal: x-y, x-m, y-m, m-n, n-z, n-o, z-o Anchor: o-F
Check Understanding

Forward direction extends a satisfying assignment into a 3-coloring; backward direction reads an assignment from any valid 3-coloring.

Try it yourself: NP-Completeness tab -> enter formula -> Build Graph -> inspect a clause gadget.

5Exact Algorithm — Backtracking

Backtracking is an exact solver: assign a color, check safety, recurse, and undo on conflict. It guarantees correctness but can grow exponentially.

Pruning Rule

Before assigning a color to a vertex, check neighbors. If any neighbor already has that color, skip immediately.

BACKTRACK(i): if i == n: return true v = orderedVertices[i] for c in {0,1,2}: if SAFE(v,c): color[v] = c if BACKTRACK(i+1): return true color[v] = null return false
Complexity Reality

Worst-case remains exponential (about O(3^n) branches), but pruning often cuts many branches in practice.

Try it yourself: Algorithms tab -> source K4 -> Build & Run Both and compare operations.

6Practical Algorithm — Greedy

Greedy scans vertices in an ordering and gives each vertex the smallest available color not used by its colored neighbors. It is fast, simple, and ordering-sensitive.

Performance Guarantee

Greedy runs in O(V+E) and uses at most Delta + 1 colors for any ordering.

What It Does Not Guarantee

Greedy does not guarantee near-optimal chi(G) on general graphs. A bad order can be much worse than optimal.

Ordering Effect

The same graph may use fewer colors under Welsh-Powell than under natural insertion order.

Try it yourself: Algorithms tab -> source C6 -> switch ordering -> rerun and compare color counts.

Common Thread

All methods here are constraint-driven. The NP verifier checks constraints, backtracking enforces them exactly through search, and greedy enforces them locally with speed-oriented decisions.

7When Greedy Fails — Crown Graph

Crown S8 is bipartite, so chi(S8)=2. But with interleaved natural order a1,b1,a2,b2,a3,b3,a4,b4, greedy can use 4 colors.

Counterexample Message

Same graph, different ordering: natural can use 4 colors while Welsh-Powell returns 2 on this exact graph.

Why This Matters

If we could always find the best ordering efficiently, we could solve hard coloring instances much more directly.

Check Understanding

The graph stays bipartite regardless of ordering; only greedy's behavior changes with the order.

Try it yourself: Design tab -> run crown counterexample under Natural and Welsh-Powell.

Appendix Hook — Exact vs Exponential

Keep in mind why these practical workflows matter: exact coloring can require exponential search, which is why heuristics and proof-guided demonstrations are central in real systems.

3-Coloring Overview

Given graph G = (V, E), decide whether vertices can be colored with three colors so adjacent vertices differ.

  • 2-coloring is polynomial (check bipartiteness via BFS).
  • 3-coloring is NP-complete.
  • Applications: register allocation, exam scheduling, frequency assignment.
